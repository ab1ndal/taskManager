Two changes, both minimal:

1. Early-return guard — add this block directly after the existing `if (!open) return null;` line (line 109):

```tsx
if (workspaces.length === 0) {
  return (
    <div
      className="fixed inset-0 z-50 flex items-center justify-center bg-black/40"
      onClick={onClose}
    >
      <div
        className="bg-[var(--color-surface)] rounded-[14px] border border-[var(--color-border)] p-6 w-full max-w-sm mx-4 shadow-xl text-center"
        onClick={(e) => e.stopPropagation()}
      >
        <p className="text-sm text-[var(--color-text-secondary)] mb-4">
          You must join a workspace before creating tasks.
        </p>
        <a
          href="/workspaces"
          className="text-sm font-medium text-[var(--color-accent)] hover:underline"
        >
          Go to Workspaces
        </a>
      </div>
    </div>
  );
}
```

2. Submit guard — in handleSubmit, change the existing early-return condition (line 85) from:

```tsx
if (!title.trim() || selectedMemberIds.length === 0) return;
```

to:

```tsx
if (!title.trim() || !workspaceId || selectedMemberIds.length === 0) return;
```

The `!workspaceId` check is a defensive fallback only — the early-return above prevents the full form from rendering when workspaces is empty, so this guard should never trigger in normal flow.
