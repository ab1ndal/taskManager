## Component Inventory

### Workspace Card (directory entry)
**Container:** `rounded-[8px] border border-[var(--color-border)] bg-[var(--color-surface)] px-4 py-3` with `--shadow-card`
- **Heading:** workspace name, 14px 500, text-primary
- **Kind Badge:** "Household" (indigo-100 bg / indigo-700 text) or "Work" (sky-100 bg / sky-700 text), 11px 500, rounded-full, px-2 py-0.5
- **Member count:** "3 members", 11px, text-muted, right-aligned
- **Action state:**
  - Unjoined: accent-colored "Join" button, 14px 500
  - Joined: green "Joined" badge, 11px 500, matching kind badge styling pattern

### Create Workspace Modal
**Trigger:** "Create Workspace" button in top-right of page header
**Form fields:**
- Workspace name (text input, required)
- Kind selector (radio buttons or dropdown: Household / Work)
- Submit button: "Create"
**Post-success:** Modal closes, toast shows, directory revalidates

### No-Workspace Banner (on /tasks page)
**Trigger:** Show if authenticated user has 0 workspace members
**Content:** "You're not in any workspace yet." + link to /workspaces with text "Browse workspaces"
**Styling:** Top of tasks content area, same card pattern as workspace cards or a distinct alert-style banner

## Interaction Patterns

### Join Flow
1. User views workspace card with "Join" button
2. Click "Join" → pending state (button opacity to 0.5 or loading spinner)
3. Server action executes (uses admin client, RLS bypass)
4. On success: button becomes "Joined" badge inline, no page reload
5. Toast: `Joined "[workspace name]"`

### Create Flow
1. Click "Create Workspace" → modal opens
2. User enters name, selects kind, clicks "Create"
3. Pending state on submit button
4. On success: modal closes, directory query revalidates
5. Toast: `Workspace "[name]" created`
6. New workspace appears in list with "Joined" badge

### Error Handling
- Join fails: inline error toast, button returns to "Join" state
- Create fails: inline error toast, modal remains open for retry
